## data镜像创建

譬如-v /data/changhao/android_agt/android_data_15:/data
给定一个目录，需要 root 权限，打包 /data/changhao/android_agt/android_data_15 到 ~/redroid_images。

1) 使用 squashfs（首选）

安装工具（Debian/Ubuntu）：

```bash
cd /data
# 生成 /root/redroid_images/android_data_11.sqsh
sudo mksquashfs . /root/redroid_images/android_data_11.sqsh -comp zstd -Xcompression-level 5
```

解包到挂载点（keeper 在创建设备时会调用 unsquashfs）：

```bash
sudo unsquashfs -f -d /root/redroid_data/1 /root/redroid_images/android_data_11.sqsh
```

或直接只读挂载（无需解包）：

```bash
sudo mount -t squashfs -o loop /root/redroid_images/android_data_11.sqsh /mnt/android_data_11
# 非 root 用户可以使用 squashfuse 挂载： squashfuse /root/redroid_images/android_data_11.sqsh /mnt/android_data_11
```

优点：不会包含指向绝对路径的符号链接、压缩率高、读性能好。`keeper.py` 已支持以 `.sqsh` 结尾的快照并使用 `unsquashfs` 解包。

2) 备选：使用 tar（当无法安装 squashfs 工具时）

创建时确保使用相对路径并展开符号链接，避免绝对链接问题：

```bash
# 在 /data 下运行，--dereference 展开符号链接，-C 保证相对路径
sudo tar -C /data --dereference -czf /root/redroid_images/android_data_11.tar.gz .
```

解包时请谨慎，keeper 中会对 tar 内容进行路径安全检查以防路径穿越。

小结：优先使用 squashfs；若受限可用 tar + --dereference 作为回退方案。
